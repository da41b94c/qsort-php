# qsort-php

Учебная реализация быстрой сортировки для PHP 8.1+.

Проект показывает:

- рекурсивное разбиение массива относительно опорного значения;
- сохранение или удаление повторяющихся значений;
- сортировку объектов и сложных структур через callback-компаратор;
- стратегию median-of-three для выбора pivot;
- автоматические тесты на PHP 8.1–8.4.

> Для прикладного production-кода обычно следует использовать встроенные `sort()`, `usort()` и другие функции PHP. Они быстрее, экономнее по памяти и не ограничены глубиной рекурсии этой учебной реализации.

## Установка

```bash
composer require da41b94c/qsort-php
```

До публикации пакета в Packagist репозиторий можно подключить через секцию `repositories` Composer или клонировать напрямую.

## Сохранение повторов

```php
<?php

require __DIR__ . '/vendor/autoload.php';

$values = [8, 99, 1, 3, 10, 199, 2, 8, 3, 99, 3];

$result = qsortWithRepeats($values);

// [1, 2, 3, 3, 3, 8, 8, 10, 99, 99, 199]
```

## Удаление повторов

```php
$result = qsort($values);

// [1, 2, 3, 8, 10, 99, 199]
```

## Универсальная функция

`quickSort()` сохраняет повторы по умолчанию и принимает callback-компаратор с тем же контрактом, что и `usort()`.

```php
$result = quickSort(
	[3, 1, 4, 2],
	static fn (int $left, int $right): int => $right <=> $left,
);

// [4, 3, 2, 1]
```

### Сортировка объектов

```php
$items = [
	(object) ['name' => 'Gamma', 'priority' => 30],
	(object) ['name' => 'Alpha', 'priority' => 10],
	(object) ['name' => 'Beta', 'priority' => 20],
];

$result = quickSort(
	$items,
	static fn (object $left, object $right): int => $left->priority <=> $right->priority,
);
```

### Удаление эквивалентных значений

При `preserveDuplicates: false` повтором считается любое значение, для которого компаратор возвращает `0`.

```php
$result = quickSort(
	$items,
	static fn (object $left, object $right): int => $left->priority <=> $right->priority,
	preserveDuplicates: false,
);
```

## Сложность и ограничения

Средняя временная сложность quick sort — `O(n log n)`, худшая — `O(n²)`. Дополнительная память требуется для новых массивов на каждом уровне рекурсии.

Вместо постоянного выбора первого элемента используется median-of-three: медиана первого, среднего и последнего значений. Это улучшает поведение на уже отсортированных и обратно отсортированных массивах, но не исключает специально подобранный худший сценарий.

Ключи исходного массива не сохраняются.

## Benchmark

Небольшой диагностический benchmark сравнивает учебную реализацию со встроенным `sort()` на случайном, отсортированном и обратно отсортированном наборах:

```bash
php benchmark/benchmark.php
```

Результаты зависят от версии PHP, процессора и текущей нагрузки. Benchmark не является критерием корректности и намеренно не запускается в CI.
